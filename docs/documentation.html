Data Flow

  1. Web App → IndexedDB: All task CRUD via Dexie with reactive useLiveQuery
  2. Web App → Graph API: MSAL token → fetch to Microsoft Graph endpoints → cache in IndexedDB
  3. Web App → AI: Vite proxy forwards to NVIDIA NIM / Anthropic → JSON action → execute locally
  4. CLI → tasks.json: Direct file read/write for all task operations
  5. CLI ↔ Web App: Web app polls localhost:3001/api/tasks every 15s, syncs both directions

// Lightweight fetch wrapper — no SDK dependency
async function graphFetch<T>(endpoint: string): Promise<T> {
  const token = await authActions.getAccessToken();
  const response = await fetch(
    `https://graph.microsoft.com/v1.0/${endpoint}`,
    { headers: { Authorization: `Bearer ${token}` } }
  );
  return response.json();
}

Token Flow

  1. User clicks "Sign in with Microsoft" → msalInstance.loginRedirect()
  2. Browser redirects to Microsoft login → user authenticates → redirects back
  3. msalInstance.handleRedirectPromise() captures the response
  4. Subsequent API calls use acquireTokenSilent() with iframe renewal
  5. If silent fails → acquireTokenRedirect() for re-authentication

8. AI Integration

Two Providers

ProviderModelCostMethodRate Limit
NVIDIA NIMmeta/llama-3.1-70b-instructFreePrompt-based JSON actions40 req/min
Anthropicclaude-sonnet-4-20250514PaidNative tool callingPer plan

How It Works (NVIDIA NIM)

  1. User types natural language message
  2. System prompt is built with current task list + available action schemas
  3. Message sent to Llama 3.1 70B via https://integrate.api.nvidia.com/v1/chat/completions
  4. AI responds with either:
    • JSON action block — parsed and executed against local DB
    • Plain text — displayed as conversational response
  5. After action execution, result sent back to AI for friendly summary

How It Works (Anthropic Claude)

  1. Same user message, but sent with tool definitions (function schemas)
  2. Claude returns tool_use content blocks with function name + parameters
  3. Tool executed locally → result sent back as tool_result
  4. Claude generates final natural language response

CORS Solution

// vite.config.ts — Proxy external APIs to avoid browser CORS
server: {
  proxy: {
    '/api/nvidia': {
      target: 'https://integrate.api.nvidia.com',
      changeOrigin: true,
      rewrite: (path) => path.replace(/^\/api\/nvidia/, ''),
    },
    '/api/claude': {
      target: 'https://api.anthropic.com',
      changeOrigin: true,
      rewrite: (path) => path.replace(/^\/api\/claude/, ''),
    },
  },
}

Fallback: Regex Command Parser

When no AI API key is configured, the chat uses a rule-based parser with 20+ regex patterns:

// Example patterns
/^(?:add|create|new)\s+(?:a\s+)?task[:\s]+(.+)/i  → add_task
/^(?:complete|done|finish)[:\s]+(.+)/i             → complete_task
/^(?:show|list)\s+(?:my\s+)?tasks?/i               → show_tasks
/^(?:calendar|meetings|events)/i                    → show_calendar

10. CLI Agent

Installation

# Install globally
cd agent && npm link

# Configure
echo "NVIDIA_API_KEY=nvapi-your-key-here" > agent/.env

# Run from anywhere
todo

Features

  • AI-powered: Natural language → NVIDIA NIM (Llama 3.1 70B) → JSON action → execute
  • Quick commands: tasks, myday, important — instant, no AI call
  • REST server: Runs on port 3001 alongside the REPL
  • Sync: Web app polls every 15s with hash-based change detection
  • Zero dependencies: Pure Node.js — node:http, node:fs, node:readline, node:crypto
  • Colored output: ANSI escape codes for task status, stars, icons

Sync Mechanism

// Web app (useAgentSync.ts):
1. Fetch GET localhost:3001/health → if fails, skip silently
2. Fetch GET localhost:3001/api/tasks → get agent tasks
3. Compare hash of task IDs + updatedAt → skip if unchanged
4. Import new tasks into IndexedDB (bulkAdd)
5. Update existing tasks where agent.updatedAt > local.updatedAt
6. Export web app tasks back to agent (POST /api/tasks/import)

15. Development Prompts & Journey

Below is the complete sequence of prompts used during the AI-assisted development of this application with Claude Opus 4.6. Each prompt built on the previous phase, progressively adding features. These prompts are enhanced versions of the originals.

"Build a clone of Microsoft To Do: Lists, Tasks & Reminders as a React + TypeScript + Vite web application. It should include full task management — create, edit, delete, complete tasks with subtasks, categories, recurrence rules, reminders, and due dates. Implement a 3-panel layout (sidebar, task list, detail panel) matching the real Microsoft To Do structure. Use Zustand for state management, Dexie.js for IndexedDB offline-first storage, dnd-kit for drag-and-drop reordering, Radix UI for accessible components, and Lucide React for icons. Include smart lists: My Day, Important, Planned, Assigned to Me, and custom lists with color themes."

Result Full To Do app with all core features, 50+ components, 5 Zustand stores, Dexie database with 5 tables.

Prompt 2 — Premium UI Overhaul

"The UI looks very old school — we need a much better, modern UI. Think glassmorphism, 3D animations, and premium design. Implement frosted glass effects with backdrop-filter blur on the sidebar, detail panel, dialogs, and context menus. Add animated mesh gradient backgrounds.
